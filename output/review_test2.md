Because this is a unit test inside `cli::mod` and it imports `use super::*;`, it can still see parent-module items through Rust’s descendant privacy rules even if the `pub use args::*;` / `pub use dispatch::run_with;` facade is broken for external callers. In other words, this test can stay green while downstream code using `changeguard::cli::{Cli, LedgerCommands, ...}` fails. If the goal is to lock the public API, this needs to be an integration test that imports from the crate boundary.
