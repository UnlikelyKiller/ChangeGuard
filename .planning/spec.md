# API Endpoint Ownership, Auth, and Consumer Graph Specification

## 1. Overview
Track W2 aims to elevate API endpoint tracking from 7/10 to 9/10 precision. This is achieved by creating a holistic graph around endpoints that models ownership, authentication boundaries, request schemas, and upstream/downstream consumers, allowing ChangeGuard to track impact on public API surfaces accurately.

## 2. Graph Relations & Models

### 2.1 Entity Kinds
- **Nodes**: `Endpoint`, `Service`, `Symbol`, `DataModel`, `SecurityBoundary`
- **Edges**:
  - `Service` `OWNS` `Endpoint`
  - `Symbol` (Handler) `HANDLES` `Endpoint`
  - `Endpoint` `AUTHENTICATES` using `Symbol` (Middleware/Guard)
  - `Endpoint` `AUTHORIZES` using `Symbol` (Middleware/Guard)
  - `Symbol` (Client) `CONSUMES` `Endpoint`
  - `Endpoint` `HANDLES` `DataModel` (Schema)
  - `Test` `COVERS` `Endpoint`

### 2.2 Property Extensions
**`NodeKind::Endpoint` Properties (in CozoDB)**:
- `auth_status`: `public`, `secured`, or `unknown`.
- `method`: HTTP method.
- `path`: Route path.
- `framework`: Actix, Axum, Rocket, etc.

**Distinguishing Unknown vs. Absent (Public)**:
- By default, an endpoint's `auth_status` is `unknown`.
- If an explicit public declaration is detected (e.g., public middleware bypass, explicit comment `// @public`), `auth_status` is `public`.
- If an auth guard/middleware is detected, `auth_status` is `secured` and `AUTHENTICATES` edges are drawn.

### 2.3 SQLite Enhancements
Extend `api_routes` and `ExtractedRoute`:
- `auth_requirements`: JSON string (e.g., `["Bearer"]` or `["Public"]`).
- `schema_refs`: JSON array of string URNs.
- `owning_service`: String URN or simple string.
- `consumers`: JSON array of identified consumer strings/URNs.

## 3. Extraction Mechanisms

### 3.1 Auth Extraction (Rust)
Enhance `src/index/languages/rust/routes.rs`:
- **Axum**: Trace `.layer(...)` and `.route_layer(middleware::from_fn(...))` calls attached to the router. Map the provided function to the endpoint's `AUTHENTICATES` relation.
- **Actix Web**: Track `.wrap(...)` calls on `App` or `Scope` and extract the middleware symbol.
- **Rocket**: Parse handler function parameters for types known to implement `FromRequest` that relate to auth (e.g., `User`, `Claims`, `AuthGuard`).

### 3.2 Schema Extraction
- Inspect handler function signatures to find JSON/Form payloads (e.g., `Json<MyPayload>`, `Form<MyForm>`).
- Map these `DataModel` structs to the endpoint via `HANDLES` (Payload).

### 3.3 Consumer Detection
- **Callsites**: Parse usages of `reqwest::Client` or `Hyper` clients where the URL/path explicitly matches an indexed `path_pattern`.
- **Config**: Scan configurations/environment settings for known API URL templates that can be linked to consumer services.

## 4. Subsystem Interactions

### 4.1 Knowledge Graph Join Strategy
1. SQLite `api_routes` acts as the source of truth for the raw AST extraction.
2. The `GraphWorker` (`src/index/graph_worker.rs`) loops over `api_routes` during indexing.
3. For each route, a `NodeKind::Endpoint` is upserted into CozoDB with URN `urn:endpoint:{method}:{path_pattern}`.
4. Corresponding edges (`OWNS`, `HANDLES`, `AUTHENTICATES`, `CONSUMES`) are generated by resolving symbol names to their URNs.

### 4.2 Impact Enrichment (`src/impact/enrichment/api.rs`)
- **Removal**: If an endpoint is missing in a newer scan, flag all connected `CONSUMES` nodes with a `HIGH` risk of contract breakage.
- **Auth Drift**: If `auth_status` changes from `public` to `secured`, or if the auth schema drops a required claim, emit a contract breakage warning.
- **Schema Changes**: Route payload model modifications transitively impact the `Endpoint` and downstream `CONSUMES` nodes.

## 5. CLI Design
Add a new top-level command `changeguard endpoints` in `src/cli.rs`:
```rust
/// View and analyze public API endpoints, ownership, auth, and consumers
Endpoints {
    /// Filter by specific service
    #[arg(long)]
    service: Option<String>,
    /// Filter by auth status (public, secured, unknown)
    #[arg(long)]
    auth: Option<String>,
    /// Show consumer list
    #[arg(long)]
    consumers: bool,
    /// Output as JSON
    #[arg(long)]
    json: bool,
}
```
*Output*: A tabular or JSON list displaying Method, Path, Owning Service, Auth Status, and Consumer Count.
