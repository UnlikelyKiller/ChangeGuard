# Specification: Track G3 Graph Ingestion & Provenance

## Goal
Implement a native Rust engine for loading `graphify` output into ChangeGuard, ensuring that every semantic edge is correctly linked to the **ChangeGuard Ledger**.

## Context
We now have a `graphify-out/graph.json` generated by the Python pipeline. ChangeGuard needs to "ingest" this file to make it queryable. Crucially, it must correlate these nodes with the `project_symbols` it already knows about to establish architectural provenance.

## Technical Details

### 1. JSON Loader (`src/index/graph_loader.rs`)
Implement a parser for the `graph.json` format:
- Handle `nodes` with `id`, `label`, and `category`.
- Handle `edges` with `source`, `target`, `relation`, and `confidence`.
- Support reading `confidence` strings ('EXTRACTED', 'INFERRED', 'AMBIGUOUS') and converting them to numeric floats (1.0, 0.7, 0.4).

### 2. Provenance Matching
During ingestion:
1.  Check if a node `id` matches an existing `project_symbol` name.
2.  If it matches, link the node to the symbol's `ledger_id`.
3.  If it doesn't match (e.g., a "Domain" or "Rationale" node from documentation), create a new node and link it to the **current** ledger transaction.

### 3. Batch Loading
Optimize ingestion using CozoDB's batch `put` capabilities to ensure even large graphs (5,000+ nodes) load in under 1 second.

## TDD Requirements
1.  **JSON Deserialization**: Test that a standard `graphify` JSON is correctly parsed into Rust structs.
2.  **Symbol Correlation**: Verify that a node named `analyze_risk()` is correctly linked to the existing symbol entry for that function.
3.  **Confidence Mapping**: Verify that 'INFERRED' edges are stored with a 0.7 confidence weight.
