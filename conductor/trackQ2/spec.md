# Track Q2: CLI Ergonomics & Subcommand Alignment

## Objective
Enhance the user experience of the ChangeGuard CLI by standardizing subcommand behaviors, specifically unifying `help` and `version` commands, and fixing argument parsing issues for the `ledger audit` command.

## Requirements
1. **`version` Subcommand**:
   - Introduce `version` as an alias that performs the exact same behavior as `--version`.
2. **Unify `help` vs `--help`**:
   - Ensure the output of `changeguard help [command]` is perfectly identical to `changeguard [command] --help`.
   - Remove minor formatting discrepancies generated by `clap`'s default `help` subcommand versus flag handling.
3. **Fix `ledger audit --entity`**:
   - Update `entity` from a positional argument to an optional flag argument (`-e`, `--entity`) across both `Commands::Audit` and `LedgerCommands::Audit` in `src/cli.rs`.

## Architecture & Implementation Strategy

### 1. Help & Version Unification (CLI Interceptor)
`clap` inherently generates slightly different output for `help` (as a subcommand) and `--help` (as a flag). To definitively unify them while preserving muscle memory:
- Disable `clap`'s built-in help subcommand globally using `#[command(disable_help_subcommand = true)]` on the main `Cli` struct in `src/cli.rs`.
- In `src/main.rs`, intercept OS arguments *before* passing them to `Cli::parse_from()`.
  - If `args[1]` is `"help"`, remove it and append `"--help"` to the arguments list. This maps `changeguard help <cmd>` to `changeguard <cmd> --help`.
  - If `args[1]` is `"version"`, replace it with `"--version"`.

### 2. Argument Parsing Fix
- In `src/cli.rs`, locate `Commands::Audit` and `LedgerCommands::Audit`.
- Apply `#[arg(short, long)]` to the `entity: Option<String>` field. This tells `clap` to treat it as an option flag (e.g., `--entity=src/main.rs`) rather than a positional argument, matching the help documentation.

## Testing Strategy
- **Manual CLI Tests**:
  - `cargo run -- version` outputs the standard version block.
  - `cargo run -- help` outputs the exact same help string as `cargo run -- --help`.
  - `cargo run -- help ledger audit` perfectly maps to `cargo run -- ledger audit --help`.
  - `cargo run -- ledger audit --entity src/main.rs` successfully parses without an "unrecognized flag" error.
  - `cargo run -- audit --entity src/main.rs` parses correctly at the top-level shortcut.
- **Unit Tests**:
  - Run `cargo test` to ensure argument manipulation doesn't break existing CLI configuration tests.
