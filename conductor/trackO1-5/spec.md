# Track O1-5: Cryptographic Provenance

## Objective
Harden the ChangeGuard Ledger so every transaction is mathematically proven to be generated by a specific developer key. This makes the ledger immune to tampering and satisfies strict audit requirements.

## Requirements
*   **Cryptography:** Add `ed25519-dalek = "2.1.1"` dependency.
*   **Key Generation:** Modify `changeguard init` to generate a local Ed25519 keypair per developer and store it securely (`~/.changeguard/keys/` or equivalent local app data directory).
*   **Transaction Signing:** 
    *   Compute a deterministic hash of the transaction metadata (ID, entity, category, summary, reason, timestamp).
    *   Sign the hash using the developer's private key.
*   **Storage:** Store the signature alongside the transaction rows in CozoDB (this requires a schema migration for the `ledger_entry` relation).
*   **Verification:** Implement `changeguard verify --signatures` to offline-verify all ledger entries against public keys.

## Definition of Done (DoD)
*   [ ] `ed25519-dalek` is integrated successfully.
*   [ ] `changeguard init` generates and stores a keypair securely.
*   [ ] `changeguard ledger commit` signs the transaction.
*   [ ] The CozoDB schema is updated to store signatures.
*   [ ] `changeguard verify --signatures` successfully validates signed transactions and fails on tampered data.
