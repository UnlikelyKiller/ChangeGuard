# Milestone S: Global Intelligence & Precision Search

This milestone outlines the strategy for evolving ChangeGuard from a change-intelligence tool into a comprehensive codebase research and precision-indexing engine, achieving parity with and exceeding the capabilities of Sourcebot.

## Objective
To implement sub-millisecond global code search, compiler-grade precision indexing, and LLM-powered semantic discovery across the entire federated codebase.

---

## Track S1: High-Performance Global Code Search (Trigram Index)

**Goal**: Implement a "grep-on-steroids" search that works instantly regardless of codebase size.

### Implementation Strategy
*   **Indexer Engine**: Integrate `tantivy` (Rust-native search engine) or implement a Trigram-based FTS index within the existing SQLite/CozoDB layer.
*   **Trigram Indexing**: Decompose every file into 3-character sequences (trigrams) to enable near-instant regex and substring matching without full-file scanning.
*   **Incremental Updates**: Hook the indexer into the `changeguard watch` and `scan` pipelines to ensure the search index is always synchronized with the working tree.

### Key Deliverables
*   `changeguard search <query>`: A CLI command that returns ranked results with syntax-highlighted snippets.
*   `--regex` and `--literal` flags for search precision.
*   Performance benchmark ensuring <50ms search time on a 1M+ line codebase.

---

## Track S2: Precise LSP-Based Indexing (SCIP/LSIF)

**Goal**: Move from "fuzzy/heuristic" Tree-sitter indexing to "compiler-perfect" precision for navigation and impact analysis.

### Implementation Strategy
*   **SCIP Ingestion**: Implement a native Rust parser for the **SCIP (Symbolic Code Index Protocol)** format.
*   **Build System Integration**: Provide a mechanism (or `changeguard doctor` check) to ingest indices generated by `scip-typescript`, `scip-python`, or `scip-rust`.
*   **Graph Enrichment**: Mirror SCIP's precise "definition-to-reference" links into the CozoDB Knowledge Graph.

### Key Deliverables
*   `changeguard index --scip <path>`: Command to ingest a high-precision index.
*   **Perfect Navigation**: Zero-ambiguity "Find References" in the `viz` and `impact` commands for projects with SCIP indices.
*   Hybrid Mode: Automatically fall back to Tree-sitter for files/languages without a SCIP index.

---

## Track S3: Semantic Discovery for "Code Snippets"

**Goal**: Enable "conceptual search" where users can find code by its purpose, not just its text.

### Implementation Strategy
*   **Fine-Grained Chunking**: Decompose source files into logical blocks (functions, classes, and cohesive logic blocks) using the Tree-sitter AST.
*   **Local Vector Embedding**: Use the local LLM (`llama-server`) to generate high-dimensional embeddings for each code block.
*   **Vector Database**: Utilize CozoDB's native vector storage or the `sqlite-vss` extension to store and query these embeddings.

### Key Deliverables
*   `changeguard ask --code "How do we handle retry logic?"`: Returns semantically related code snippets even if the word "retry" isn't in the function name.
*   **Conceptual Clustering**: Group "similar logic" in the `viz` graph, allowing users to see duplicated or near-duplicate logic across the codebase.
*   **Snippet RAG**: Automatically pull relevant code snippets into the context for `changeguard ask` suggestions.

---

## Track S4: Unified Search & Audit Portal

**Goal**: Combine the Ledger, Search, and Graph into a single cohesive research interface.

### Implementation Strategy
*   **Unified Query Parser**: A single search entry point that routes to the Ledger (history), the Trigram Index (code), and the KG (architecture).
*   **Federated Search**: Extend search to query sibling repositories via the existing ChangeGuard federation layer.

### Key Deliverables
*   `changeguard audit --global <query>`: A comprehensive search that returns a timeline of relevant ledger entries, code changes, and architectural shifts.
*   Enhanced `viz` with a "Search & Filter" sidebar for real-time graph navigation.

---

## Definition of Done (DoD)
1. [ ] Global search completes in <100ms on 10,000+ files.
2. [ ] SCIP ingestion results in 100% precision for cross-file references.
3. [ ] Semantic search can find a function based on a conceptual description.
4. [ ] All search commands support the `--json` flag for integration with external tools.
5. [ ] The `changeguard doctor` command verifies the health of the Trigram and Vector indices.
